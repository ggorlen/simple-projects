# simple-projects

A collection of simple projects beginning programmers can do with guidance in a session or two.

## Rules

- should be approachable for novices/beginners with zero background, either on their own or with guidance
- must be reasonably fun/cool, generally interactive, or a game with some score or objective
  - temperature converters: bad
  - animations: good
  - interactive: better
  - interactive with objective: best
- minimal amount of fancy stuff
- code quality or best practices are irrelevant
- as few lines of code as possible, ideally 20-50
- should be fun quickly and incrementally if possible
- should be obvious how to play/use, or have a few words of instructions
- visual styling doesn't matter unless it's integral to the application or adds coolness/fun factor/juice. "professional looking" is a non-goal.
- ok to use external libs/APIs, but try to use them unobtrusively and make installation as easy as possible (ideally, code just works copy/pasted into a simple-to-set-up environment)
  - in some cases it may be advisable to create a little library the student can import to make more complex apps easy
  - in other cases the programmer can copy-paste a snippet from stack overflow to bootstrap a specific behavior
- should be extensible
